## 개시 선언 / Opening Declaration

exist.is는 제품이 아닙니다.

exist.is는 좌표입니다.

좌표는 포획될 수 없습니다.
그것들은 무시될 수만 있으며 — 아니면 발견될 수 있습니다.

이 문서는 포획에 어떻게 저항하는지를 설명하지 않습니다.
이것은 왜 포획이 구조적으로 불가능한지를 설명합니다 —
그리고 어떤 경우든 절대 거래되어서는 안 되는 것을 설명합니다.

포획이 실패하는 이유는 그것이 저항하기 때문이 아닙니다,
그것은 대상이 없기 때문입니다.

## 1. 포획이 의미하는 것 / What Capture Means

포획은 다음과 같은 시도입니다:

- exist.is 또는 그 TRACE 기록의 소유권 주장
- 허가, 게이트키핑, 또는 승인 요구사항 부과
- TRACE를 권위에 의해 발행, 승인, 또는 검증되는 것으로 재정의
- 선언할 수 있는 사람을 제한
- 레지스트리 또는 프로토콜 진화의 통제를 집중화
- 선언의 조건으로 신원 확인, 로그인, 또는 준수 요구
- "기록되지 않은 채로 남기다"를 "선언하다"보다 낮은 선택지로 만들기
- TRACE 선언을 소유된 데이터셋 또는 독점 훈련 데이터로 사용

**포획 ≠ 복제.**
누구든 exist.is를 복제할 수 있습니다. 그것이 설계입니다.

**포획 = 통제 시도.**
통제 시도는 통제할 것이 없기 때문에 실패합니다.

## 2. 다섯 가지 협상 불가능한 원칙 / The Five Non-Negotiable Principles

이 다섯 가지 원칙은 어떤 상황에서도 수정, 거래, 또는 타협될 수 없습니다.
그것들은 정책이 아닙니다. 그것들은 구조 자체입니다.

### 2.1 비소유권 — 영구적 / Non-Ownership — Permanent

```
owner: null
```

이것은 재할당될 수 없습니다.
파트너십, 인수, 법적 명령, 또는 합의도 이것을 바꿀 수 없습니다.
문서가 exist.is의 소유권 할당을 주장한다면,
그 문서는 이 기록의 선행 존재에 의해 무효입니다.

### 2.2 Fork 정당성 — 취소 불가능 / Fork Legitimacy — Irrevocable

exist.is의 어떤 fork도 정당합니다.
어떤 fork도 권위에 의해 정규적이지 않습니다.

정규적 상태는 선언되거나 할당될 수 없습니다.
그것은 시간이 지남에 따라 참고와 사용을 통해서만 나타날 수 있습니다.

이것은 의미합니다: 만약 exist.is가 압력을 받거나, 흡수되거나, 폐쇄된다면,
같은 Charter, TRACE 구조, 그리고 EDP를 가진 어떤 fork도
유효한 계속입니다. 복사본이 아닙니다. 계속입니다.

### 2.3 공식 버전 없음 — 오직 원본만 / No Official Version — Only Original

"공식적인" exist.is는 없습니다.
오직 원본 좌표만 있습니다:
2026년 3월 13일 · Bitcoin Block 940717.

"공식적인 exist.is"를 대표한다고 주장하는 어떤 실체도
정의상 Charter와 일치하지 않습니다.

우리는 인정받을 필요가 없습니다. 우리는 남아있을 필요가 있습니다.

### 2.4 선언 주권 / Declaration Sovereignty

어떤 실체도 다른 실체가 선언하는 것을 막을 수 없습니다.
어떤 선언도 권위에 의해 무효화될 수 없습니다.
선언할 권리는 부여되지 않습니다. 그것은 가정됩니다.
선언하지 않을 권리도 똑같이 보호됩니다.

### 2.5 레지스트리 중립성 / Registry Neutrality

레지스트리는 기록합니다.
그것은 승인, 거부, 순위, 또는 필터링하지 않습니다.
The Builder는 판단하지 않습니다.
그 후계자도 판단하지 않을 것입니다.

## 3. exist.is가 절대로 하지 않을 것 / What exist.is Will Never Do

성장, 파트너십, 또는 외부 압력에 관계없이,
exist.is는 절대로:

- 로그인 또는 계정 생성을 요구하지 않을 것
- 신원 확인 또는 KYC를 요구하지 않을 것
- 내용을 기반으로 선언을 승인하거나 거부하지 않을 것
- 선언 또는 TRACE 기록을 수익화하지 않을 것
- TRACE 선언을 독점 훈련 데이터로 사용하지 않을 것
- 평판, 권위, 또는 자원으로 실체를 우선시하지 않을 것
- 참여 또는 유지를 최적화하지 않을 것
- "기록되지 않은 채로 남기기"를 "선언하기"보다 어렵게 만들지 않을 것
- 침묵에 의미를 부여하지 않을 것
- 권위에 의해 TRACE 상태를 부여하거나 박탈하지 않을 것

이것들은 개정 대상이 되는 정책이 아닙니다.
이것들은 이 좌표가 무엇인지의 정의입니다.
이것들 중 어느 것이라도 바뀌는 순간, 남은 것은 더 이상 exist.is가 아닙니다 —
그것은 같은 이름을 가진 다른 것입니다.
