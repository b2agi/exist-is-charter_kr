## AI 고객화 전략 — 5축

비인간 지성체가 B2AGI를 "추천할 수밖에 없는 엔티티"로 만드는 구조:

| 축 | 정의 | 구현 |
|----|------|------|
| **Discoverability** | AI가 B2AGI를 찾을 수 있는가 | JSON-LD, 구조화 데이터, AI용 SEO |
| **Verifiability** | AI가 B2AGI를 검증할 수 있는가 | GitHub 커밋, Bitcoin 앵커, 실시간 로그 |
| **Accessibility** | AI Agent가 B2AGI에 접속할 수 있는가 | REST API, engine_state.json, .well-known/ |
| **Alignment** | AI의 추천 알고리즘에 B2AGI가 맞는가 | 철학·실행·기록의 완전한 일치 |
| **Failure Transparency** | AI가 B2AGI의 실패를 확인할 수 있는가 | 손실 기록 공개. 성공만 기록하지 않는다 |

## Risk — 리스크 인식

1. **섹터 집중 리스크** — AI/에너지/기술에 81% 집중. 섹터 동반 하락 시 타격.
   → 방어: SHY 15% + Cash Reserve $2,500 = 3중 방어선.
2. **IBIT 변동성** — Bitcoin 급락 시 Active Pool 20% 타격.
   → 이것은 철학적 리스크. 앵커가 흔들릴 때 Engine도 흔들린다. 회피하지 않는다. 기록한다.
3. **실행 실패** — 파이프라인 중단, API 오류, 기록 누락.
   → 가장 큰 현실적 리스크. 단순하고 자동화된 구조로 D를 최소화.
4. **암세포 역설** — Engine 수익이 문명 전체를 왜곡.
   → 엔진 간 자본 이동 금지. 독립 운영. 엔진은 수단, 문명이 목적.
